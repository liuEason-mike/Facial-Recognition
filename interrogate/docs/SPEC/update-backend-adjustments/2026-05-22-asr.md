# ASR 审讯关键词提炼功能规范

日期：2026-05-22
范围：`interrogate` 后端关键词提炼接口与 `html-interrogate` 前端 ASR 展示链路。

## 1. 背景

当前 `html-interrogate` 前端通过项目后端 `/ws/asr` 推送麦克风音频，后端再代理 DashScope 公有实时 ASR 模型并返回中文转写文本。现需在不影响实时转写展示的前提下，基于累计转写文本抽取审讯场景关键词，并在前端可视化展示。

## 2. 目标

1. 前端继续保持低延迟 ASR 文本展示。
2. 当前端累计新增有效文本达到约 100 字时，触发一次关键词提炼。
3. 后端接收本轮累计文本，构建提示词，并调用大模型接口抽取关键词。
4. 关键词聚焦审讯场景，至少覆盖人物、时间、地点、行为、事件五类。
5. 后端返回结构化 JSON，前端按类别分色渲染关键词。
6. 大模型接口不可用或未配置时，后端提供规则兜底，保证功能可演示。

## 3. 一期实现链路

```text
浏览器麦克风
  -> /ws/asr
  -> DashScope 公有实时 ASR
  -> 前端实时转写
  -> 前端累计约 100 字
  -> POST /api/asr/keywords/extract
  -> 后端构建提示词
  -> 大模型接口（DashScope compatible 或兼容接口）
  -> 后端清洗 JSON
  -> 前端关键词可视化
```

实时 ASR 音频主链路必须走项目后端 `/ws/asr`，由后端统一维护 DashScope 公有模型鉴权、协议转换、结果归一化和入库边界。前端不得硬编码外部 ASR WebSocket 地址。

## 4. 文本累计与触发规则

- 前端只累计非空 ASR 文本。
- 去除空白字符后按字符数统计，达到 `100` 字触发。
- 连续重复文本不重复计入缓冲，降低 ASR partial 反复修正造成的重复触发。
- 每次触发后清空当前缓冲，开始下一轮累计。
- 触发接口调用不阻塞实时 ASR 展示。
- 接口失败只记录运行日志，不中断音频采集。

## 5. 后端接口

### 5.1 提取关键词

```http
POST /api/asr/keywords/extract
Content-Type: application/json
```

请求体：

```json
{
  "session_id": "20260522001",
  "suspect_id": "1",
  "window_id": "asr-keywords-1",
  "text": "累计到 100 字左右的 ASR 转写文本",
  "context": "可选，上一次关键词或摘要"
}
```

成功响应：

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "window_id": "asr-keywords-1",
    "provider": "llm",
    "keywords": [
      {
        "text": "前往上海",
        "category": "地点",
        "confidence": 0.91,
        "source": "凌晨三四点前往上海",
        "count": 1
      }
    ]
  }
}
```

错误响应：

```json
{
  "code": 40001,
  "msg": "text_required",
  "data": null
}
```

## 6. 关键词提炼规则

大模型提示词必须约束：

- 只基于输入文本抽取，不编造事实。
- 输出 JSON，不输出 Markdown。
- 关键词为短语，避免单字、语气词和无意义短句。
- 分类限定为：`人物`、`时间`、`地点`、`行为`、`事件`。
- 合并重复表达，最多返回 15 个关键词。
- 优先保留审讯价值更高的名词短语和动作短语。

示例关键词：

- `前往上海`
- `凌晨三四点`
- `周某`
- `签署仓储合同`
- `预付款转出`

## 7. 前端展示

- 在 ASR 转写面板内新增“关键词提炼”区块。
- 按类别分色显示关键词标签。
- 显示关键词提炼状态：待累计、提炼中、提炼失败。
- 标签展示文本使用 `text` 字段。
- 发生接口错误时保留已有关键词，不清空展示。

## 8. 配置

后端通过环境变量读取大模型配置，不在代码或文档中写真实密钥：

| 变量 | 说明 |
| --- | --- |
| `DASHSCOPE_API_KEY` | DashScope 访问密钥，推荐用于默认接入 |
| `KEYWORD_LLM_ENDPOINT` | 大模型 HTTP 接口地址，默认 `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` |
| `KEYWORD_LLM_API_KEY` | 大模型访问密钥，优先级高于 `DASHSCOPE_API_KEY` |
| `KEYWORD_LLM_MODEL` | 模型名称，默认 `qwen3-max` |
| `KEYWORD_LLM_TIMEOUT` | 请求超时时间，默认 12 秒 |

兼容 `DASHSCOPE_LLM_ENDPOINT`、`DASHSCOPE_LLM_MODEL` 和 `BAIYING_LLM_ENDPOINT`、`BAIYING_LLM_API_KEY`、`BAIYING_LLM_MODEL` 命名。

## 9. 验收标准

1. 后端服务能从模型 JSON 或规则兜底结果中返回结构化关键词。
2. `POST /api/asr/keywords/extract` 对空文本返回 `40001`。
3. 前端累计文本达到 100 字后触发关键词请求。
4. 前端展示关键词标签，并按类别分色。
5. `pnpm test:unit`、`pnpm typecheck`、`pnpm build` 通过。
6. 相关后端 pytest 通过。
