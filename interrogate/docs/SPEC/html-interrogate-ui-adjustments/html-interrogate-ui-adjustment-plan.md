# html-interrogate 讯问室页面呈现调整方案

日期：2026-05-19
范围：`interrogate/html-interrogate` 前端页面，不修改后端接口。

## 目标

本次调整围绕讯问室页面的实时分析呈现做减法和聚焦：

1. 视频画面只展示五类目标标注：人脸、眉毛、瞳孔、鼻子、嘴巴。
2. 标注样式统一为绿色细框，不展示其他区域、点位或骨架线。
3. 左侧分析面板移除“最近回包”，不再显示视频 time 和 ASR time。
4. 后端返回的 AU 强度数据完整展示，不再只截取前 6 项。
5. 设备状态中的视频链路、音频链路、视频帧、音频包改为更紧凑的展示样式。

## 实现设计

### 五类视频标注

`src/utils/faceRegions.ts` 继续负责把后端 `region` 转成可绘制的框，但输出范围收敛到五类业务目标：

- `face`：显示为“人脸”。
- `left_eyebrow`、`right_eyebrow`：显示为“眉毛”。
- `left_pupil`、`right_pupil`：显示为“瞳孔”。
- `nose`：显示为“鼻子”。
- `mouth`：显示为“嘴巴”。

保留左右眉毛、左右瞳孔的独立框，是因为后端返回的坐标天然就是左右拆分；页面文案仍归类为五类。

### AU 强度

新增前端工具函数统一读取后端 AU 数据：

- 优先展示 `au_52` 的全部字段。
- 如果没有 `au_52`，回退展示 `au_intensities` 的全部字段。
- 展示时按强度从高到低排序，不再截断。

### 面板精简

`FaceAnalysisPanel.vue` 调整：

- 删除“最近回包”区块。
- 删除 `lastFaceMessageAt`、`lastAsrMessageAt` 展示依赖。
- 设备状态采用小型四宫格，压缩高度和字号。
- AU 列表采用紧凑滚动区，避免 52 项把面板撑得过长。

## 验收

1. 单元测试覆盖五类标注过滤和完整 AU 列表排序。
2. `pnpm test:unit` 通过。
3. `pnpm typecheck` 通过。
4. `pnpm build` 通过。
