请修改前端关键词提炼交互逻辑：

1. 关键词列表展示要求

前端关键词提炼列表中，关键词需要展示为具体可识别内容，而不是泛化名称。

例如：
- 银行卡：展示具体卡号形式，如 6222 1235 8888 1234
- 车牌号：展示具体车牌号，如 沪A·12345
- 手机号：展示具体手机号形式，如 13845875678

关键词列表中可以存在以下类型：
- 银行卡
- 车牌号
- 手机号

每个关键词需要携带类型字段，例如：
- bank_card
- vehicle_plate
- phone_number

前端根据关键词类型决定点击后打开的查询结果页面内容。

2. 点击关键词后的交互要求

用户点击关键词后，不要立即展示结果。

需要先在关键词位置或页面中展示“查询中...”状态，模拟正在通过专业渠道查询。

查询中状态持续 1–2 秒。

查询中状态示例：
- 查询中...
- 正在调取专业数据源...
- 正在核验信息...

1–2 秒后，再从页面右侧滑出查询结果浮框。

3. 右侧浮框展示要求

查询结果浮框从页面右侧滑出，形成右侧抽屉式浮层。

浮框宽度不要过大，占屏幕高度的 1/3 不到，宽度为：目前的浮框宽度

同一时间只允许打开一个浮框。

点击关闭按钮、遮罩层或按 Esc 可以关闭浮框。

4. 查询结果真实性要求

右侧浮框不是简单 JSON 展示，而是模拟“专业渠道查询结果”的业务页面视觉效果。

需要包含：
- 标题
- 查询状态
- 数据来源
- 查询编号
- 查询时间
- 信息字段
- 风险等级或核验结果

整体风格应类似公安、运营商、车辆、金融等专业系统查询结果，而不是普通弹窗。

5. 不同关键词类型对应不同查询结果页面

不同类型关键词点击后，必须展示不同的页面内容。

不要共用同一个静态标题和静态内容。

6. 前端状态逻辑要求

点击关键词后的状态流程：

未点击状态：
关键词正常展示。

点击后：
当前关键词进入 loading 状态，显示“查询中...”。

1–2 秒后：
关闭 loading 状态，打开右侧查询结果浮框。

再次点击其他关键词：
关闭当前浮框，重新进入新的关键词查询中状态，然后展示新的查询结果浮框。

7. 页面交互效果要求

右侧浮框需要有滑入动画。

动画方向：
从右向左滑入。

关闭时：
从左向右滑出。

浮框内容需要分区展示：
- 顶部标题区
- 查询状态区
- 数据来源和查询编号区
- 核心信息字段区
- 底部核验结果区

8. 注意事项

所有查询结果均为模拟数据，仅用于前端演示。

不要真实调用银行、运营商、车辆系统接口。

不要把结果写成 JSON 文本展示。

不要多个关键词共用同一个查询结果页面。

不同类型关键词必须展示不同标题、不同字段和不同查询结果。
