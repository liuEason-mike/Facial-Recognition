# html-interrogate 关键词详情浮框需求

版本：v0.1
更新日期：2026-05-25
适用范围：`html-interrogate` 讯问室页面中 ASR 关键词提炼结果的前端展示与交互。

## 1. 背景

当前讯问室右侧转写区域会展示 ASR 关键词提炼结果。为便于审讯人员快速查看重点对象信息，需要将部分特定类型关键词升级为可点击状态，并在页面右侧展示固定模拟详情。

## 2. 目标

- 当关键词类型命中指定范围时，关键词标签展示为可点击状态。
- 点击可点击关键词后，从页面右侧滑出详情浮框。
- 浮框覆盖在当前讯问室页面上方，不跳转路由，不刷新页面。
- 浮框展示与关键词类型对应的固定模拟信息。
- 浮框整体视觉需与当前审讯室页面风格一致。

## 3. 非目标

- 本次不接入真实外部查询接口。
- 本次不新增后端接口、数据库表或持久化逻辑。
- 本次不改变 ASR 关键词提炼接口返回结构。
- 本次不改变非指定类型关键词的展示和交互。

## 4. 触发范围

仅以下关键词类型展示为可点击状态：

| 类型 | 展示行为 | 点击后详情 |
| --- | --- | --- |
| 身份证 | 可点击 | 展示身份证模拟查询结果 |
| 手机号 | 可点击 | 展示手机号模拟查询结果 |
| 车牌号 | 可点击 | 展示车牌号模拟查询结果 |
| 车次 | 可点击 | 展示车次模拟查询结果 |

非以上四类关键词保持原有展示，不可点击，不触发浮框。

## 5. 交互要求

### 5.1 打开浮框

- 用户点击可点击关键词后，右侧浮框从页面右侧滑出。
- 浮框应覆盖在当前页面上方，页面主体内容保持在原位。
- 同一时间只展示一个浮框。
- 当浮框已打开时，点击另一个可点击关键词，应直接切换浮框内容为新关键词对应类型。

### 5.2 关闭浮框

以下任一操作可关闭浮框：

- 点击浮框关闭按钮。
- 点击遮罩层。
- 按 `Esc` 键。

关闭后，页面恢复到浮框打开前的交互状态。

### 5.3 键盘与可访问性

- 可点击关键词应使用按钮语义或等效可访问语义。
- 浮框关闭按钮应有明确可访问名称。
- 浮框打开时应提供 `Esc` 关闭能力。
- 不可点击关键词不应使用按钮样式或按钮语义。

## 7. 样式要求

- 浮框整体风格与当前审讯室页面保持一致，使用现有深色背景、边框、阴影、主题色和状态色。
- 遮罩层应能弱化页面背景，但不应造成页面内容错位。
- 可点击关键词应比普通关键词更明显，但不能破坏当前关键词列表的信息密度。
- 浮框内容使用字段名与字段值成组展示，空值不应出现。
- 移动端或窄屏下浮框宽度应自适应，不应超出视口或遮挡关闭按钮。

## 8. 验收标准

- 四类指定关键词展示为可点击状态，非指定类型关键词不可点击。
- 点击四类指定关键词后，右侧浮框能打开并展示对应固定模拟数据。
- 连续点击不同指定关键词时，浮框只保留一个，并切换为最后一次点击的关键词类型。
- 点击关闭按钮、遮罩层或按 `Esc` 均能关闭浮框。
- 浮框打开和关闭不影响 ASR 转写、关键词提炼、音频波形、视频画面和微表情分析的实时展示。
- 页面通过前端单元测试、类型检查和构建验证。

## 10. 后续实现建议

- 在前端维护可点击关键词类型白名单。
- 将固定模拟数据放在组件内常量或独立工具文件中，避免散落在模板里。
- 抽屉打开状态、当前关键词类型和关闭事件应由页面组件或关键词面板内部统一管理。
- 若后续接入真实接口，应先新增接口契约文档，再替换固定模拟数据来源。
