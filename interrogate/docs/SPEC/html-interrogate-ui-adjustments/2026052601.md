请修改前端“关键词提炼”模块的交互逻辑和右侧查询浮框展示效果，要求如下：

一、关键词列表展示要求

1. 关键词列表中不要展示泛化名称，而是展示具体可识别内容。

示例：
- 银行卡：6222 2913 2913 1234
- 车牌号：沪A·12345
- 手机号：13829135678

2. 每个关键词对象需要包含类型字段，用于判断点击后打开哪类查询页面。

关键词类型如下：
- bank_card：银行卡
- vehicle_plate：车牌号
- phone_number：手机号

示例数据结构：

{
  label: "6222 2913 2913 1234",
  type: "bank_card"
}

{
  label: "沪A·12345",
  type: "vehicle_plate"
}

{
  label: "13829135678",
  type: "phone_number"
}

前端根据 type 字段决定右侧浮框展示的查询来源、标题、字段和结果内容。

二、点击关键词后的状态流程

用户点击关键词后，不要立即展示最终查询结果。

点击后的完整流程如下：

1. 用户点击关键词。
2. 如果当前已经打开了右侧浮框，需要先关闭当前浮框。
3. 当前关键词进入查询状态。
4. 右侧浮框先滑出“查询中”页面。
5. 查询中页面持续 1–2 秒。
6. 1–2 秒后，查询中页面切换为对应类型的“查询结果”页面。
7. 同一时间只允许存在一个右侧浮框。

注意：
查询中不是在关键词位置简单显示文字，而是右侧浮框先打开一个“查询中页面”。
也就是说右侧浮框存在两个状态页面：
- 查询中页面
- 查询结果页面

三、查询中页面要求

点击关键词后，右侧浮框先从右侧滑入，展示查询中页面。

查询中页面需要根据关键词类型展示不同的专业来源。

查询中页面视觉要求：
- 不要只显示普通文本
- 需要有 loading 图标或扫描线动画
- 需要展示当前查询平台名称
- 需要展示“查询中...”或“正在核验信息...”
- 整体风格模拟专业系统查询过程

四、右侧浮框尺寸要求

当前右侧浮框高度铺满了整个屏幕，需要修改。

新的浮框要求：

1. 浮框从页面右侧滑出，形成右侧抽屉式浮层。
2. 浮框宽度保持目前已有宽度，不要扩大。
3. 浮框高度不要铺满整个屏幕。
4. 浮框高度控制在屏幕高度的 1/3 以内。
5. 建议高度设置为：
   - height: 32vh;
   - max-height: 360px;
   - min-height: 280px;
6. 浮框垂直方向建议居中显示，或者距离顶部 20% 左右显示。
7. 不要做成全高 drawer。

示例样式方向：

.query-drawer {
  position: fixed;
  right: 0;
  top: 20%;
  width: 当前已有宽度;
  height: 32vh;
  max-height: 360px;
  min-height: 280px;
  transform: translateX(100%);
  transition: transform 0.3s ease;
}

.query-drawer.open {
  transform: translateX(0);
}

五、查询结果页面要求

查询完成后，右侧浮框内容从“查询中页面”切换为“查询结果页面”。

查询结果页面不是 JSON 展示，而是模拟专业渠道查询结果页面。

页面必须包含以下区域：

1. 顶部标题区
2. 查询状态区
3. 数据来源和查询编号区
4. 核心信息字段区
5. 底部核验结果区

整体视觉风格应类似：
- 公安查询系统
- 运营商实名查询系统
- 金融账户核验系统
- 专业业务后台查询页面

不要显示原始 JSON。
不要所有类型共用一个标题。
不要所有类型共用同一套字段。

七、交互细节要求

1. 点击银行卡关键词：
先打开右侧浮框查询中页面，显示“正在调取资金管控平台...”
1–2 秒后切换为“银行卡信息核验”结果页面。

2. 点击手机号关键词：
先打开右侧浮框查询中页面，显示“正在调取开普勒云砚...”
1–2 秒后切换为“手机号实名核验”结果页面。

3. 点击车牌号关键词：
先打开右侧浮框查询中页面，显示“正在调取公安部云搜索...”
1–2 秒后切换为“车辆信息查询”结果页面。

4. 再次点击其他关键词：
关闭当前浮框或重置当前浮框内容。
重新进入新的查询中状态。
1–2 秒后展示新的查询结果。

5. 点击关闭按钮、遮罩层或按 Esc 可以关闭浮框。

6. 关闭浮框后，需要清空当前 loading 状态和当前选中关键词状态。

八、动画要求

1. 浮框打开：
从右向左滑入。

2. 浮框关闭：
从左向右滑出。

3. 查询中到查询结果：
可以使用淡入淡出或内容切换动画。

4. 不要出现多个浮框叠加。
5. 不要点击多个关键词后同时触发多个定时器导致结果错乱。
6. 点击新关键词时，需要清除上一个 setTimeout。

十、注意事项

1. 所有查询结果均为前端模拟数据，仅用于演示。
2. 不要真实调用银行、运营商、车辆系统接口。
3. 不要把结果展示成 JSON 文本。
4. 不同类型关键词必须展示不同标题、不同数据来源、不同字段和不同结果。
5. 查询中页面和查询结果页面是两个不同状态页面。
6. 右侧浮框高度必须控制在屏幕 1/3 以内，不能再铺满全屏高度，展示查询的数据可紧凑展示。
7. 浮框宽度保持当前已有宽度。
