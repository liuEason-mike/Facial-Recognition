# 产品需求文档（PRD）

版本：v0.1
更新日期：2026-04-25
适用范围：`interrogate` 审讯实时分析系统。

## 1. 产品定位

`interrogate` 是面向审讯现场的 AI 辅助分析系统。系统通过浏览器采集摄像头和麦克风数据，经由后端实时分析面部表情、动作单元、头姿、视线、注意力、心率异常和音频转写，为审讯操作员提供实时观察、风险提示和过程记录。

本系统只提供辅助分析和过程展示，不作为审讯结论、司法事实认定或心理状态判定的唯一依据。

## 2. 目标用户

| 用户 | 目标 |
| --- | --- |
| 审讯操作员 | 在审讯过程中实时查看视频分析、ASR 转写、风险提示和运行状态 |
| 研判人员 | 基于实时记录和异常事件辅助复盘审讯过程 |
| 运维人员 | 部署、监控和维护后端服务、数据库、模型文件与外部依赖 |
| 开发人员 / AI 代理 | 按 PRD 与工程 SPEC 修改功能，避免破坏现有协议和部署约束 |

## 3. 首版范围

首版以 `html-interrogate/` 审讯实时工作台和后端 WebSocket 分析链路为核心。

必须支持：

- 浏览器打开新前端工作台。
- 操作员输入或确认嫌疑人编号。
- 一键开始审讯，同时启动摄像头、麦克风、`/ws/face` 和 `/ws/asr`。
- 实时展示视频画面、人脸区域、情绪指标、心率、注意力、异常事件和 ASR 转写。
- 一键停止审讯，释放媒体设备、定时器和 WebSocket 连接。
- 后端将实时情绪、异常评分和 ASR 文本写入 MySQL。
- 通过 `GET /health` 检查服务存活。

首版不优先支持：

- 完整案件管理、嫌疑人管理、历史复盘和权限系统。
- 审讯结论自动生成。
- 多房间、多操作员协同审讯。
- GPU/NPU 专用推理部署。
- 将前端风险提示包装为确定性判断。

## 4. 核心业务流程

1. 操作员打开 `html-interrogate/` 工作台。
2. 操作员确认 `suspect_id`，点击“开始审讯”。
3. 前端申请摄像头和麦克风权限。
4. 前端连接 `ws://<host>:5000/ws/face`，按帧发送 JPEG Base64。
5. 前端连接 `ws://<host>:5000/ws/asr`，发送 16k 单声道 PCM S16LE Base64 音频包。
6. 后端返回视频分析结果和 ASR 透传结果。
7. 前端持续展示指标、转写文本、风险事件和运行日志。
8. 操作员点击“停止审讯”，前端发送 ASR 结束包并释放资源。
9. 后端持久化实时帧、异常评分和 ASR 文本。

## 5. 功能需求

| 编号 | 需求 |
| --- | --- |
| PRD-F01 | 新前端默认入口为 `html-interrogate/`，旧前端 `html/` 保持稳定，未明确要求不得修改 |
| PRD-F02 | 后端统一监听 `5000` 端口，对外提供 HTTP 与 WebSocket 服务 |
| PRD-F03 | `/ws/face` 接收 `{id,image,test_status}`，默认 `test_status=0` 实时检测 |
| PRD-F04 | `/ws/asr` 接收 `{type,audio,seq,suspect_id}`，停止时发送 `end=true` 结束包 |
| PRD-F05 | WebSocket 错误响应统一为 `{code,msg,data}` |
| PRD-F06 | 前端必须清晰展示视频连接、音频连接、权限、重连、错误和停止状态 |
| PRD-F07 | 风险提示必须以辅助提示呈现，不得表达为最终审讯结论 |
| PRD-F08 | 生产数据库目标为 MySQL 8.x，生产环境禁止依赖 SQLite fallback |
| PRD-F09 | 日志不得输出密钥、完整数据库连接串、图像 Base64 或音频 Base64 |

## 6. 验收标准

- `python run.py` 启动后，`GET http://127.0.0.1:5000/health` 返回 `code=0`。
- 本地新前端通过 `html-interrogate/` 的 Vite 服务访问，默认地址为 `http://localhost:5173/`。
- `/ws/face` 和 `/ws/asr` 均通过 `5000` 端口访问。
- 向 WebSocket 发送非法 JSON 时，返回 `{code:40001,msg:"invalid_json",data:null}`。
- 开始审讯后，页面可以同时启动视频采集、音频采集、面部分析连接和 ASR 连接。
- 停止审讯后，页面释放摄像头、麦克风、定时器和 WebSocket 连接。
- 修改 `html-interrogate/` 后至少运行 `pnpm typecheck` 和 `pnpm build`，无法运行时必须说明原因。

- 是否为历史复盘、案件管理和嫌疑人档案建立独立 PRD。
