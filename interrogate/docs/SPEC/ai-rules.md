# AI 生成代码规则

AI 修改本项目代码时必须遵守以下规则：

- 修改前必须先读取相关代码，不允许凭空假设项目结构。
- 新需求必须先写入或更新 `docs/SPEC/PRD.md`，工程约束必须同步更新 `docs/SPEC/` 下的对应 spec 子文档。
- 不得修改 `.env`、`.env_pro`、生产密钥、数据库密码，除非用户明确要求。
- 不得提交模型权重、日志、数据库文件、生成的 `.pkl` 模型、音视频大文件。
- 后端新增接口必须补充请求格式、响应格式、错误码和日志行为。
- 后端数据库变更必须同步更新 ORM 模型和 MySQL DDL，并说明迁移方式。
- WebSocket 协议变更必须同步更新前端 TypeScript 类型。
- 前端新增环境变量必须添加到 `vite-env.d.ts` 并在本 spec 中记录。
- Python 代码应保持 service、route、repository、model 分层，不把业务逻辑写入路由函数。
- AI / CV 推理代码不得在主请求路径中重复加载大模型，必须使用可复用实例或明确缓存策略。
- 日志不得输出 Base64 图像、Base64 音频、密钥、完整数据库连接串。
- Docker 变更必须说明基础镜像、系统依赖、启动命令、端口和构建上下文。
- 所有改动完成后必须至少运行对应的静态检查、单元测试或启动验证；无法运行时必须说明原因。
- 任何自动修复不得覆盖用户已有未提交变更。
- 每次代码变更后，不需要任何相关 git 变更操作，仅需在 spec 中记录，我审查后再提交。
- 后端新增或修改 HTTP 接口必须遵循项目统一响应结构；WebSocket 错误响应必须使用统一 `{code,msg,data}` 结构。
- 现有 `/ws/face` 成功响应当前为裸 EmotionFrame，现有 `/ws/asr` 成功响应当前为外部 ASR 透传结构；若改为统一成功响应结构，必须同步更新 `docs/api/`、前端 TypeScript 类型和页面解析逻辑。
