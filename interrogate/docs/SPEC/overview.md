# 工程规范总览

版本：v0.2
更新日期：2026-04-22
适用范围：`interrogate` 后端、前端、数据库、镜像、部署与 AI 生成代码约束。

## 项目目标

`interrogate` 是面向审讯场景的 AI 辅助分析系统。系统通过浏览器采集摄像头帧和麦克风音频，经由 Flask WebSocket 后端完成实时面部分析、情绪识别、AU 动作单元分析、头姿与视线分析、注意力判断、心率估计、异常检测和 ASR 转写，并将关键结果持久化到数据库。

工程目标：

- 提供统一的审讯实时分析服务，后端监听 `5000` 端口。
- 前端以 Vue 3 大屏方式展示视频、指标、异常提示、转写文本和辅助审讯信息。
- 后端以 WebSocket 支撑实时视频帧和音频流处理。
- 数据库以 MySQL 为生产目标，保存实时情绪、异常评分和音频转写记录。
- Docker 镜像支持前后端一体化交付。
- 所有后续代码变更必须同时满足本工程 spec 和 PRD 的产品目标。

## 文档职责

- PRD 定义产品目标、用户场景和功能需求。
- spec 定义工程约束、接口协议、数据库、构建、部署、运维和 AI 生成代码规则。
- 新需求必须先写入或更新 `docs/SPEC/PRD.md`，工程约束必须同步更新 `docs/SPEC/` 下的对应 spec 子文档。
